# windows_winget_package (Data Source)

Reads the state of a single winget-managed package on a remote Windows host
via WinRM and the `Microsoft.WinGet.Client` PowerShell module.

If the package is absent from both the winget catalog **and** the Windows
ARP (Add/Remove Programs) registry, the data source returns a Terraform
error â€” non-existent packages must fail loudly. There is no silent empty
result.

The `Microsoft.WinGet.Client` module **must** already be installed on the
target host. The provider does **not** auto-install it. Install it once
with:

```powershell
Install-Module Microsoft.WinGet.Client -Scope AllUsers -Force
```

~> **Elevated session required.** The WinRM session must run as a Local
Administrator (same requirement as the twin resource).

## Example Usage

```terraform
# Look up an installed winget-managed package on the target Windows host.
data "windows_winget_package" "vscode" {
  package_id = "Microsoft.VisualStudioCode"
  source     = "winget"
}

output "winget_package_info" {
  value = data.windows_winget_package.vscode
}

# Microsoft Store package.
data "windows_winget_package" "wt" {
  package_id = "9N0DX20HK701"
  source     = "msstore"
}

# ARP-only package (legacy MSI/EXE visible to winget without a source).
# Set source = "" to look up Add/Remove Programs entries directly.
data "windows_winget_package" "legacy" {
  package_id = "Contoso.LegacyApp"
  source     = ""
}
```

## Error Reference

The data source surfaces the same typed errors as the twin
[`windows_winget_package`](../resources/winget_package.md) resource Read
path, plus a dedicated `not_found` diagnostic:

| Kind | Terraform diagnostic |
|------|----------------------|
| `not_found` | `package_id` is absent from both the winget catalog and ARP. Data sources never silently produce empty state. |
| `module_missing` | `Microsoft.WinGet.Client` is not installed on the target host. |
| `source_unreachable` | Network error reaching the winget source (retried once after 5 s, then surfaced). |
| `multiple_matches` | `Get-WinGetPackage` returned more than one row even after the client-side `Equals(Id)` filter. |
| `unknown` | Unexpected PowerShell error or WinRM transport failure (propagated verbatim â€” never masked as `not_found`). |

## Permissions Required

- Local Administrator on the target Windows host (member of
  `BUILTIN\Administrators`).
- WinRM enabled and reachable; the user must be permitted in the
  `PSSessionConfiguration` (default: `Microsoft.PowerShell`). HTTPS is
  preferred.
- `Microsoft.WinGet.Client` PowerShell module installed on the target host
  (`Scope AllUsers`).
- Network access from the target host to the configured winget source
  (`winget` or `msstore`). Not required when `source = ""` (ARP-only).
