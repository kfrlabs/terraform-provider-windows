# windows_environment_variable (Data Source)

Reads a single Windows environment variable (`machine` or `user` scope) from a
remote host via WinRM + PowerShell.

Returns the stored value **verbatim and unexpanded** (raw `%VAR%` tokens) and
the registry kind (`expand`). If the variable does not exist, the data source
returns a Terraform error (non-existent variables must fail loudly — there is
no silent empty result).

## Example Usage

```terraform
# Look up an existing machine-scope variable.
data "windows_environment_variable" "java_home" {
  name  = "JAVA_HOME"
  scope = "machine"
}

output "java_home" {
  value = data.windows_environment_variable.java_home.value
}

# Look up a REG_EXPAND_SZ variable — value contains raw %VAR% tokens.
data "windows_environment_variable" "tools" {
  name  = "TOOLS_DIR"
  scope = "machine"
}

output "tools_is_expandable" {
  value = data.windows_environment_variable.tools.expand
}
```

## Schema

### Required

- `name` (String) Windows environment variable name to look up. Must be
  non-empty and must not contain `=`.
- `scope` (String) Scope to look up: `"machine"` or `"user"`.

### Read-Only

- `expand` (Boolean) `true` when the registry value kind is `REG_EXPAND_SZ`;
  `false` when `REG_SZ`.
- `id` (String) Composite ID `"<scope>:<name>"`. Populated by the provider.
- `value` (String) Verbatim value as stored in the registry (unexpanded
  `%VAR%` tokens).
