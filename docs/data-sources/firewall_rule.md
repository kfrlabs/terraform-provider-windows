# windows_firewall_rule (Data Source)

Reads the observed state of a Windows Defender Firewall with Advanced Security
rule on a remote Windows host via WinRM and PowerShell (`NetSecurity` module).

The rule is looked up by its **stable technical `name`** (InstanceID), which
is distinct from `display_name`. This is the same identifier used by the
`windows_firewall_rule` resource and by `Get-NetFirewallRule -Name`.

This data source does **not** manage the rule lifecycle (no create, update, or
delete). Returns a Terraform error when the rule is not found in the target
`policy_store` (defaults to `PersistentStore`).

## Example Usage

```terraform
# Read a built-in firewall rule.
data "windows_firewall_rule" "file_sharing" {
  name = "FPS-NB_Session-In-TCP"
}

output "file_sharing_enabled" {
  value = data.windows_firewall_rule.file_sharing.enabled
}

# Inspect a rule managed by a sibling resource.
data "windows_firewall_rule" "managed" {
  name = windows_firewall_rule.allow_app.name
}

# Look up a rule in a non-default policy store.
data "windows_firewall_rule" "gp_rule" {
  name         = "GroupPolicy-Inbound-RDP"
  policy_store = "GroupPolicy"
}
```

## Schema

### Required

- `name` (String) Stable technical identifier of the firewall rule
  (InstanceID) to look up.

### Optional

- `policy_store` (String) Policy store the rule lives in. Defaults to
  `PersistentStore`. One of: `PersistentStore`, `ActiveStore`, `GroupPolicy`,
  `RSOP`, `SystemDefaults`, `StaticServiceStore`, `ConfigurableServiceStore`.

### Read-Only

- `id` (String) Data source ID; equal to the technical firewall rule Name
  (InstanceID).
- `display_name` (String) Human-readable name shown in the Windows Firewall UI.
- `description` (String) Free-form description of the rule.
- `enabled` (Boolean) Whether the rule is active.
- `direction` (String) Traffic direction (`Inbound` or `Outbound`).
- `action` (String) Action taken on matching traffic (`Allow`, `Block`,
  `NotConfigured`).
- `profile` (List of String) Network profiles the rule applies to.
- `edge_traversal_policy` (String) Edge-traversal behaviour.
- `group` (String) Rule group (RuleGroup).
- `protocol` (String) IP protocol keyword (`TCP`, `UDP`, `ICMPv4`, `ICMPv6`,
  `IGMP`, `Any`) or numeric 0-255.
- `local_port` (List of String) Local TCP/UDP port(s).
- `remote_port` (List of String) Remote TCP/UDP port(s).
- `local_address` (List of String) Local IP addresses, CIDR ranges, or address
  keywords.
- `remote_address` (List of String) Remote IP addresses, CIDR ranges, or
  address keywords.
- `program` (String) Full path to the matched program executable, or `Any`.
- `service` (String) Short name of the matched Windows service, or `Any`.
- `interface_type` (String) Interface type (`Any`, `Wireless`, `Wired`,
  `RemoteAccess`).
