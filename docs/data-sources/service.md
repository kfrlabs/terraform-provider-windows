# windows_service (Data Source)

Reads the observed state of a Windows service without managing its lifecycle.

Write-only attributes (`status`, `service_password`) are intentionally absent.

Returns an error when the named service is not found in the Windows Service
Control Manager (SCM) on the target host.

## Example Usage

```terraform
# Look up the Windows Update service by its short name.
data "windows_service" "wuauserv" {
  name = "wuauserv"
}

output "wuauserv_display_name" {
  value = data.windows_service.wuauserv.display_name
}

output "wuauserv_start_type" {
  value = data.windows_service.wuauserv.start_type
}

output "wuauserv_current_status" {
  value = data.windows_service.wuauserv.current_status
}
```

### Required

- `name` (String) Short name of the Windows service to look up.

### Read-Only

- `id` (String) Data source ID; equal to the Windows short service name.
- `display_name` (String) Human-readable display name shown in `services.msc`.
- `description` (String) Textual description of the service.
- `binary_path` (String) Full path to the service executable including any arguments.
- `start_type` (String) Service start mode: `Automatic`, `AutomaticDelayedStart`, `Manual`, or `Disabled`.
- `current_status` (String) Observed runtime state: `Running`, `Stopped`, or `Paused`.
- `service_account` (String) Account under which the service runs (e.g. `LocalSystem`, `NT AUTHORITY\NetworkService`).
- `dependencies` (List of String) Ordered list of short service names this service depends on.
