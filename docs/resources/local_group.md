# windows_local_group (Resource)

Manages a Windows local group (Local Users and Groups) on a remote host via
WinRM and PowerShell (`Microsoft.PowerShell.LocalAccounts` module, built-in
from **Windows Server 2016 / Windows 10** onwards).

This resource manages **only the group entity itself** — name, description, and
SID. Membership management is explicitly out of scope for v1 and is delegated
to the future `windows_local_group_member` resource, following the
`aws_iam_group` / `aws_iam_group_membership` split pattern (ADR-LG-3).

~> **ID anchored on SID.** The Terraform resource ID equals the group's
**Security Identifier** (e.g. `S-1-5-21-…-1001`). The SID is assigned by
Windows at creation time and **does not change when the group is renamed**. A
change to the `name` attribute issues `Rename-LocalGroup -SID <sid>` in place
— no resource replacement (ADR-LG-1).

~> **Local groups only.** This resource manages Windows **local** groups via
`Microsoft.PowerShell.LocalAccounts`. Domain groups visible in Active Directory
Users & Computers are **not** manageable through this resource. Domain-joined
machines are fully supported for *local* group management (EC-9).

~> **Built-in group protection.** Attempting `terraform destroy` on a built-in
group (SID prefix `S-1-5-32-*`, e.g. Administrators, Users, Guests) results in
a **hard error** — not a warning. Built-in groups can be *managed* (name,
description) through this resource but never destroyed (ADR-LG-2, EC-2).

## Example Usage

### Minimal — create a custom local group

```terraform
resource "windows_local_group" "app_admins" {
  name = "AppAdmins"
}
```

### With description

```terraform
resource "windows_local_group" "app_admins" {
  name        = "AppAdmins"
  description = "Administrators of the AppSuite application stack."
}
```

### Full provider block + group lifecycle

```terraform
terraform {
  required_providers {
    windows = {
      source  = "kfrlabs/windows"
      version = "~> 0.1"
    }
  }
}

provider "windows" {
  host      = var.windows_host
  username  = var.windows_username
  password  = var.windows_password
  auth_type = "ntlm"
}

# Create a custom local group.  The resource ID (id) is set to the group SID
# so it remains stable even if the name is later changed in HCL.
resource "windows_local_group" "app_admins" {
  name        = "AppAdmins"
  description = "Administrators of the AppSuite application stack."
}

output "app_admins_sid" {
  value       = windows_local_group.app_admins.sid
  description = "SID of the AppAdmins group — stable across renames."
}
```

### In-place rename (no resource replacement)

```terraform
# Renaming from "AppAdmins" to "ApplicationAdmins" issues
# Rename-LocalGroup -SID <sid> -NewName "ApplicationAdmins".
# The SID (and resource ID) remain unchanged.
resource "windows_local_group" "app_admins" {
  name        = "ApplicationAdmins"
  description = "Administrators of the AppSuite application stack."
}
```

## Schema

### Required

- `name` (String) Name of the local group (e.g. `AppAdmins`).

  Constraints (EC-6):
  - Length: 1..256 characters
  - Forbidden characters: `/` `\` `[` `]` `:` `;` `|` `=` `,` `+` `*` `?` `<` `>` `"`
  - Must not consist solely of whitespace characters
  - Must not start or end with a space

  **In-place rename:** a change in this attribute issues
  `Rename-LocalGroup -SID <sid> -NewName <new_name>` — no resource replacement
  (ADR-LG-1, EC-5).

  **Case normalisation:** drift comparison uses `strings.EqualFold` so a
  case-only difference does not trigger a spurious rename. State always stores
  the casing returned by Windows (ADR-LG-4, EC-4).

### Optional

- `description` (String) Optional free-text description of the group.
  Windows caps this at **256 characters** (EC-7). An empty string (`""`) is
  valid and represents no description. Defaults to `""` when omitted in HCL.

### Read-Only

- `id` (String) Terraform resource ID. Equal to `sid` (the group Security
  Identifier). **Stable across renames**: renaming the group via the `name`
  attribute does not change the resource ID (ADR-LG-1).
- `sid` (String) Security Identifier (SID) of the group
  (e.g. `S-1-5-21-…-1001`). Assigned by Windows when `New-LocalGroup`
  completes. Stable across renames. Used as the canonical Terraform resource
  ID. Read-only computed attribute.

## Error Classification

Errors returned by the underlying PowerShell calls are classified into stable
kinds, surfaced verbatim in the diagnostic detail under `Kind:`:

| Kind                | Typical cause                                                                               |
|---------------------|---------------------------------------------------------------------------------------------|
| `not_found`         | Group disappeared outside Terraform (drift, EC-3) or import target does not exist (EC-10). |
| `already_exists`    | A group with the same name already exists at Create time; use `terraform import` (EC-1).   |
| `builtin_group`     | Attempt to destroy a BUILTIN group (SID prefix `S-1-5-32-*`, EC-2).                        |
| `permission_denied` | The WinRM user lacks Local Administrator rights on the target host (EC-8).                 |
| `name_conflict`     | The rename target name is already used by another local group (EC-5).                      |
| `invalid_name`      | Windows-side name validation failure — defence-in-depth after schema validators (EC-6).    |
| `unknown`           | Catch-all for unexpected PowerShell or WinRM transport failures.                           |

## Permissions

- **Local Administrator** on the target Windows host (required for `New-`,
  `Set-`, `Rename-`, `Remove-LocalGroup`).
- Standard authenticated WinRM user is sufficient for `Get-LocalGroup`
  (read-only operations).
- WinRM enabled and reachable from the Terraform runner host:
  - HTTP: TCP 5985
  - HTTPS: TCP 5986
- Windows Server 2016 / Windows 10 minimum
  (`Microsoft.PowerShell.LocalAccounts` is built-in from these versions).

## Import

A `windows_local_group` resource can be imported using **either the group name
or its SID** (EC-10, ADR-LG-6). The import ID is auto-detected:

- If the value **starts with `S-`** → treated as a SID (`Get-LocalGroup -SID <value>`).
- Otherwise → treated as a group name (`Get-LocalGroup -Name <value>`).

After a successful import, the resource ID is always set to the group SID
regardless of which import path was used.

### Import by group name

```shell
terraform import windows_local_group.app_admins AppAdmins
```

### Import by SID

```shell
terraform import windows_local_group.app_admins S-1-5-21-1234567890-987654321-1122334455-1001
```

Retrieve the SID of a group on the target host with:

```powershell
(Get-LocalGroup -Name "AppAdmins").SID.Value
```
