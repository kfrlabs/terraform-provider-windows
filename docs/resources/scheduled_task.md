# windows_scheduled_task (Resource)

Manages a Windows Scheduled Task via WinRM and PowerShell (ScheduledTasks
module, requires Windows Server 2012 / Windows 8 or later).

Key behaviours:

- Task folders (`path`) are created recursively on Create and pruned on
  Delete (empty-folder-only prune).
- `name` and `path` are **ForceNew** — changing either destroys and
  recreates the task.
- Passwords are **write-only** (sent to Windows, never read back). Increment
  `password_wo_version` to rotate a credential without replacing the task.
- `OnEvent` triggers use raw task-XML injection (ADR-ST-5) because the
  ScheduledTasks PS module cannot set the `Subscription` property directly.
- The composite ID is `<TaskPath><TaskName>`, e.g. `\MyFolder\MyTask`.

~> **Note — write-only password.** `principal.password` is transmitted to
Windows on create/update but is **never** read back from the host. Rotating
the password out-of-band will not appear as drift.

~> **ForceNew attributes.** Changing `name` or `path` destroys and recreates
the task.

## Example Usage

### Minimal — daily SYSTEM task

```terraform
resource "windows_scheduled_task" "backup" {
  name        = "Daily-Backup"
  path        = "\\"
  description = "Nightly backup script managed by Terraform."
  enabled     = true

  actions {
    execute   = "C:\\Scripts\\backup.ps1"
    arguments = "-Destination D:\\Backups"
  }

  triggers {
    type           = "Daily"
    start_boundary = "2026-01-01T02:00:00Z"
    days_interval  = 1
  }
}
```

### Advanced — weekly domain-account task with multiple actions

```terraform
resource "windows_scheduled_task" "reports" {
  name        = "Weekly-Reports"
  path        = "\\Company\\Finance\\"
  description = "Weekly report generation — Terraform-managed."
  enabled     = true

  principal {
    user_id             = "CORP\\svc-reports"
    logon_type          = "Password"
    password            = var.svc_reports_password
    password_wo_version = 1
    run_level           = "Highest"
  }

  actions {
    execute   = "C:\\Reports\\generate.exe"
    arguments = "--weekly --output C:\\Reports\\out"
  }

  actions {
    execute   = "C:\\Scripts\\archive.ps1"
    arguments = "-Source C:\\Reports\\out -Dest \\\\nas\\reports"
  }

  triggers {
    type           = "Weekly"
    start_boundary = "2026-01-05T06:00:00Z"
    days_of_week   = ["Monday"]
    weeks_interval = 1
  }

  settings {
    start_when_available           = true
    execution_time_limit           = "PT4H"
    disallow_start_if_on_batteries = false
    stop_if_going_on_batteries     = false
    multiple_instances             = "IgnoreNew"
  }
}
```

### OnEvent trigger — react to an Event Log entry

```terraform
resource "windows_scheduled_task" "disk_alert" {
  name    = "Disk-Full-Alert"
  path    = "\\Monitoring\\"
  enabled = true

  actions {
    execute   = "powershell.exe"
    arguments = "-File C:\\Scripts\\notify.ps1"
  }

  triggers {
    type = "OnEvent"
    subscription = <<-XML
      <QueryList>
        <Query Id="0" Path="System">
          <Select Path="System">*[System[EventID=2013]]</Select>
        </Query>
      </QueryList>
    XML
  }
}
```

## Import

Import a scheduled task using its composite ID `<TaskPath><TaskName>`.

```shell
# Root-folder task:
terraform import windows_scheduled_task.backup '\Daily-Backup'

# Sub-folder task:
terraform import windows_scheduled_task.reports '\Company\Finance\Weekly-Reports'
```

~> **Note** Write-only attributes (`principal.password`, `principal.password_wo_version`)
are not populated on import. Set them in your configuration and run
`terraform apply` to push the credentials.
