# windows_registry_value (Resource)

Manages a single named value (or the unnamed **Default** value) inside a Windows
registry key on a remote host via WinRM + PowerShell.

All registry operations use the `.NET Microsoft.Win32.Registry` API directly
(not `Get-ItemProperty` / `Set-ItemProperty`), providing deterministic type
control, full `REG_QWORD` range support, and correct handling of binary values
with leading zero bytes.

**Parent key auto-creation:** missing parent keys are created automatically and
recursively at Create time (analogous to `mkdir -p`). At Delete time only the
targeted value is removed — parent keys and their other values are never
touched.

~> **ForceNew attributes.** Changing `hive`, `path`, `name`, or `type`
destroys and recreates the resource. A type change (`REG_SZ` → `REG_DWORD`
etc.) intentionally requires explicit destroy + recreate to prevent silent data
corruption.

~> **REG_EXPAND_SZ and `expand_environment_variables`.** By default
(`expand_environment_variables = false`), the provider stores and reads back
the raw `%VAR%` token for stable drift detection. Setting `true` makes Read
return the expanded path and may cause perpetual plan diffs if the expansion
changes; use with caution.

## Example Usage

### REG_SZ — plain string

```terraform
resource "windows_registry_value" "app_version" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  name         = "Version"
  type         = "REG_SZ"
  value_string = "2.5.1"
}
```

### REG_EXPAND_SZ — string with environment variable expansion

```terraform
resource "windows_registry_value" "install_dir" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  name         = "InstallDir"
  type         = "REG_EXPAND_SZ"
  value_string = "%ProgramFiles%\\MyApp"
  # expand_environment_variables = false  # default: return raw %VAR% token
}
```

### REG_MULTI_SZ — multi-string list

```terraform
resource "windows_registry_value" "search_paths" {
  hive          = "HKLM"
  path          = "SOFTWARE\\MyApp"
  name          = "SearchPaths"
  type          = "REG_MULTI_SZ"
  value_strings = ["C:\\data", "D:\\share", "\\\\server\\archive"]
}
```

### REG_DWORD — 32-bit unsigned integer (as decimal string)

```terraform
resource "windows_registry_value" "max_retries" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  name         = "MaxRetries"
  type         = "REG_DWORD"
  value_string = "5"
}
```

### REG_QWORD — 64-bit unsigned integer (as decimal string)

```terraform
resource "windows_registry_value" "cache_size" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  name         = "CacheSizeBytes"
  type         = "REG_QWORD"
  value_string = "10737418240"
}
```

### REG_BINARY — raw bytes as lowercase hex

```terraform
resource "windows_registry_value" "blob" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  name         = "BinaryConfig"
  type         = "REG_BINARY"
  value_binary = "deadbeef0102"
}
```

### REG_NONE — typeless data (empty or raw bytes)

```terraform
resource "windows_registry_value" "marker" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  name         = "Marker"
  type         = "REG_NONE"
  value_binary = ""
}
```

### Default value (unnamed, `name = ""`)

```terraform
resource "windows_registry_value" "default_val" {
  hive         = "HKLM"
  path         = "SOFTWARE\\MyApp"
  # name omitted => defaults to "" (the Default value)
  type         = "REG_SZ"
  value_string = "DefaultContent"
}
```

## Schema

### Required

- `hive` (String) Registry hive (root key). One of: `HKLM`, `HKCU`, `HKCR`,
  `HKU`, `HKCC`. Input is case-insensitive; normalised to uppercase in plan
  and state. **ForceNew.**
- `path` (String) Subkey path under the hive, using backslash as the segment
  separator (e.g. `SOFTWARE\Microsoft\Windows\CurrentVersion\Run`). Must not
  include a hive prefix, must not begin or end with a backslash, and must not
  contain NUL bytes. **ForceNew.**
- `type` (String) Windows registry value type. One of: `REG_SZ`,
  `REG_EXPAND_SZ`, `REG_MULTI_SZ`, `REG_DWORD`, `REG_QWORD`, `REG_BINARY`,
  `REG_NONE`. Case-sensitive. **ForceNew.**

### Optional

- `name` (String) Value name within the key. Set to `""` (or omit) to target
  the **Default** (unnamed) value. Must not contain NUL bytes. **ForceNew.**
  Default: `""`.
- `value_string` (String) String payload. Required and non-empty for
  `REG_SZ` and `REG_EXPAND_SZ`. For `REG_DWORD`, a decimal string in the
  range `[0, 4294967295]`. For `REG_QWORD`, a decimal string in the range
  `[0, 18446744073709551615]`. Must not be set for other types.
- `value_strings` (List of String) Multi-string payload. Required for
  `REG_MULTI_SZ`. Use `[]` for an empty multi-string value. Must not be set
  for other types.
- `value_binary` (String) Binary payload as a **lowercase** hexadecimal
  string without separators (e.g. `"deadbeef"`). Required for `REG_BINARY`.
  Optional for `REG_NONE` (defaults to `""`). Must have an even number of
  characters. Must not be set for other types.
- `expand_environment_variables` (Boolean) When `true`, Read returns the
  expanded form of a `REG_EXPAND_SZ` value. **Only valid with
  `type = "REG_EXPAND_SZ"`**; a plan-time error is raised for any other type.
  Use with caution — may cause perpetual plan diffs if the expansion changes.
  Default: `false`.

### Read-Only

- `id` (String) Composite resource ID: `HIVE\PATH\NAME`
  (e.g. `HKLM\SOFTWARE\MyApp\Version`). For the Default value (`name = ""`),
  ends with a trailing backslash: `HKLM\SOFTWARE\MyApp\`.

## Cross-field Validation

The following rules are enforced at plan time:

| Rule | Condition |
|------|-----------|
| CV-1 | `value_string` required and non-empty for `REG_SZ` / `REG_EXPAND_SZ` |
| CV-2 | `value_string` decimal uint32 `[0, 4294967295]` for `REG_DWORD` |
| CV-3 | `value_string` decimal uint64 `[0, 18446744073709551615]` for `REG_QWORD` |
| CV-4 | `value_strings` required (may be `[]`) for `REG_MULTI_SZ` |
| CV-5 | `value_binary` required for `REG_BINARY` |
| CV-6 | `value_binary` optional for `REG_NONE` (`""` = zero-length byte array) |
| CV-7 | `expand_environment_variables = true` only valid with `REG_EXPAND_SZ` |

Exactly one of `value_string`, `value_strings`, or `value_binary` must be set
appropriate to the declared `type`; setting the wrong attribute is rejected at
plan time.

## Notes

### DWORD and QWORD as decimal strings

`REG_DWORD` (uint32, max `4294967295`) and `REG_QWORD` (uint64, max
`18446744073709551615`) values are represented as **decimal strings**. This
avoids integer overflow issues: uint32 max exceeds the positive range of
`int32`, and uint64 max exceeds the precision of a JSON `float64`. The
PowerShell layer casts to the correct .NET numeric type before writing to the
registry.

### REG_BINARY encoding

`value_binary` uses lowercase hex without any separator or prefix:

```terraform
# regedit shows: de ad be ef 01 02
value_binary = "deadbeef0102"
```

An empty string `""` represents a zero-length byte array (valid for both
`REG_BINARY` and `REG_NONE`). The provider normalises all reads to lowercase
hex for stable drift detection regardless of how the bytes were stored.

### Default value (`name = ""`)

The Windows **Default** value (shown as `(Default)` in regedit) has an empty
string as its name. Omit `name` or set it to `""` to target it:

```terraform
resource "windows_registry_value" "default" {
  hive         = "HKCU"
  path         = "SOFTWARE\\MyApp"
  type         = "REG_SZ"
  value_string = "hello"
}
```

The resource ID ends with a trailing backslash: `HKCU\SOFTWARE\MyApp\`.

### expand_environment_variables

When `false` (default), `REG_EXPAND_SZ` values are read verbatim using
`RegistryValueOptions.DoNotExpandEnvironmentNames`. The raw `%SystemRoot%`
token is stored in state and compared against the configuration, providing
stable drift detection.

When `true`, the provider reads the expanded form (e.g. `C:\Windows`). This
is useful for inspecting the resolved path but **may cause perpetual plan diffs**
if the expansion changes (e.g. after an OS reinstall to a different drive letter).

Write operations always use the raw `value_string` — the flag only affects reads.

### Permissions

- `HKCU` — available to any authenticated user.
- `HKLM`, `HKCR`, `HKU`, `HKCC` — require **Local Administrator** on the
  target host.

### Type conflict guard

If a registry value exists with a **different type** than declared, the
provider returns an error rather than silently overwriting the data. To adopt
an existing value of a different type: import it first, then plan a ForceNew
type change.

## Import

A `windows_registry_value` resource can be imported using its composite ID:

```
HIVE\PATH\NAME
```

```shell
# Named value
terraform import windows_registry_value.example 'HKLM\SOFTWARE\MyApp\Version'

# Default value (name="") — trailing backslash required
terraform import windows_registry_value.default 'HKLM\SOFTWARE\MyApp\'
```

~> **Trailing backslash for Default value.** When importing the unnamed Default
value the import ID must end with a trailing backslash (`HIVE\PATH\`). Quote
the argument to prevent shell backslash interpretation.

After import, run `terraform plan` to verify alignment between the declared
configuration and the live registry state. `value_string`, `value_strings`, and
`value_binary` are populated on the first refresh/plan following import.

~> **Value names containing backslashes** cannot be expressed in this ID format
(backslash is the separator). This is a known v1 limitation; such names are
exceptionally rare in practice.
