# windows_feature (Resource)

Manages installation and uninstallation of a Windows Server role or feature on
a remote host via WinRM and PowerShell. The resource is backed by
`Get-WindowsFeature`, `Install-WindowsFeature` and `Uninstall-WindowsFeature`
from the `ServerManager` module. It supports sub-features, management tools,
an offline payload `source` (SxS / WIM) and reports pending-reboot state.

~> **Windows Server only.** The `ServerManager` cmdlets ship with Windows
Server SKUs. On Windows client editions use
`Enable-WindowsOptionalFeature` / `Disable-WindowsOptionalFeature` instead;
this provider returns an `unsupported_sku` error if the cmdlets are missing.

~> **ForceNew attributes.** Changing `name`, `include_sub_features` or
`include_management_tools` destroys and recreates the resource because
`Install-WindowsFeature` cannot retroactively shrink the feature tree.

~> **Reboot semantics.** When the cmdlet reports `RestartNeeded=Yes` and
`restart` is `false`, the provider emits a Terraform warning diagnostic and
sets `restart_pending = true` instead of failing. Set `restart = true` to let
the cmdlet reboot the host automatically.

## Example Usage

### Minimal

```terraform
resource "windows_feature" "iis" {
  name = "Web-Server"
}
```

### Full role (sub-features + management tools, auto-reboot)

```terraform
resource "windows_feature" "ad_ds" {
  name                     = "AD-Domain-Services"
  include_sub_features     = true
  include_management_tools = true
  restart                  = true
}
```

### Offline source (feature payload removed from image)

```terraform
resource "windows_feature" "netfx3" {
  name   = "NET-Framework-Core"
  source = "\\\\fileserver\\share\\sources\\sxs"
}
```

## Schema

### Required

- `name` (String) Technical name of the Windows feature
  (e.g. `Web-Server`, `DNS`, `RSAT-AD-PowerShell`). Must match
  `^[A-Za-z0-9][A-Za-z0-9._-]*$`. ForceNew.

### Optional

- `include_sub_features` (Boolean) Install all sub-features
  (`-IncludeAllSubFeature`). Default `false`. ForceNew.
- `include_management_tools` (Boolean) Install management tools
  (`-IncludeManagementTools`). Default `false`. ForceNew.
- `source` (String) Optional SxS / WIM source path used when the feature
  payload has been removed (`-Source`). Required when current
  `install_state` is `Removed`.
- `restart` (Boolean) Allow `Install-WindowsFeature` /
  `Uninstall-WindowsFeature` to reboot the host automatically when needed
  (`-Restart`). Default `false`.

### Read-Only

- `id` (String) Resource identifier; equals the feature short name.
- `display_name` (String) Human-readable display name reported by
  `Get-WindowsFeature`.
- `description` (String) Description string returned by
  `Get-WindowsFeature`.
- `installed` (Boolean) `true` when `install_state` is `Installed`.
- `install_state` (String) Current install state: `Installed`, `Available`
  or `Removed`.
- `restart_pending` (Boolean) `true` when the last operation reported
  `RestartNeeded=Yes` or the OS exposes a pending-reboot flag in the
  registry.

## Error classification

Errors returned by the underlying PowerShell cmdlets are classified into
stable kinds, surfaced verbatim in the diagnostic detail under `Kind:`:

| Kind                  | Typical cause                                                  |
|-----------------------|----------------------------------------------------------------|
| `not_found`           | The feature name is unknown to the target host.                |
| `source_missing`      | `install_state=Removed` and no `source` was provided.          |
| `dependency_missing`  | A required parent / dependency feature is not installed.       |
| `permission_denied`   | The WinRM user is not Local Administrator on the target host.  |
| `unsupported_sku`     | The `ServerManager` module is not present (client SKU).        |
| `timeout`             | The WinRM call was cancelled or exceeded the provider timeout. |
| `invalid_parameter`   | Empty / malformed feature name or argument.                    |

## Permissions

- Local Administrator on the target Windows Server.
- WinRM enabled and reachable (HTTP/5985 or HTTPS/5986).
- Read access from the target host to the `source` path when provided.

## Import

A `windows_feature` resource can be imported using the short feature `name`:

```shell
terraform import windows_feature.iis Web-Server
```
