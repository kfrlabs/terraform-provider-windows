# windows_hostname (Resource)

Manages the NetBIOS computer name of a remote Windows host over WinRM and
PowerShell, backed by `Rename-Computer` plus a defensive read of
`Win32_ComputerSystem` and three `HKLM` registry keys
(`ActiveComputerName`, `ComputerName\\ComputerName`, `Cryptography\\MachineGuid`).

~> **Asynchronous rename.** `Rename-Computer` only persists the new name to
the registry; the change becomes active after the next reboot. This resource
**never reboots the host on its own** — it surfaces `pending_name` and
`reboot_pending` so an operator (or a downstream `null_resource` /
`windows_reboot` module) can orchestrate the reboot.

~> **Workgroup only (v1).** Domain-joined hosts are rejected at runtime with
the `domain_joined` error kind. A future release will add
`-DomainCredential` support.

~> **ID anchored on MachineGuid.** The Terraform resource ID equals the
target's `HKLM:\\SOFTWARE\\Microsoft\\Cryptography\\MachineGuid`. The ID
survives renames and changes only when the underlying machine is replaced —
in which case the resource is removed from state and re-created.

~> **Destroy is a no-op.** A Windows host cannot exist without a hostname.
`terraform destroy` only removes the resource from state; the active
computer name is left unchanged and a warning is emitted.

## Example Usage

### Minimal

```terraform
resource "windows_hostname" "this" {
  name  = "WEB01"
  force = true
}
```

### With reboot orchestration via null_resource

```terraform
resource "windows_hostname" "this" {
  name  = "WEB01"
  force = true
}

resource "null_resource" "reboot" {
  count = windows_hostname.this.reboot_pending ? 1 : 0

  triggers = {
    pending_name = windows_hostname.this.pending_name
  }

  provisioner "local-exec" {
    command = "echo 'Reboot ${var.windows_host} to activate rename to ${windows_hostname.this.pending_name}'"
  }
}
```

## Schema

### Required

- `name` (String) Target NetBIOS computer name.
  - length 1..15 characters
  - alphanumeric + hyphen only
  - cannot start or end with a hyphen
  - cannot be purely numeric

  Comparison against `current_name` / `pending_name` is
  **case-insensitive** (idempotent re-apply).

### Optional

- `force` (Boolean) When `true`, passes `-Force` to `Rename-Computer` to
  suppress the interactive confirmation prompt. Default `false`.

### Read-Only

- `id` (String) Terraform resource ID. Equal to `machine_id`
  (`HKLM:\\SOFTWARE\\Microsoft\\Cryptography\\MachineGuid`). Stable across
  renames.
- `machine_id` (String) Stable per-machine identifier read from the registry.
  Anchors the Terraform ID and detects machine replacement out-of-band.
- `current_name` (String) Active computer name as exposed by
  `Win32_ComputerSystem.Name`. May differ from `name` while a rename is
  pending a reboot.
- `pending_name` (String) Hostname queued to take effect on next reboot.
  Equal to `current_name` when no rename is pending. Drift comparison for
  `name` is performed against `pending_name` to avoid an apply loop while a
  reboot is pending.
- `reboot_pending` (Boolean) `true` when `pending_name` differs from
  `current_name` (case-insensitive).

## Error classification

Errors returned by the underlying PowerShell calls are classified into
stable kinds, surfaced verbatim in the diagnostic detail under `Kind:`:

| Kind                      | Typical cause                                                                  |
|---------------------------|--------------------------------------------------------------------------------|
| `invalid_name`            | The desired `name` fails the NetBIOS rule (length, charset, purely numeric).   |
| `permission_denied`       | The WinRM user lacks Local Administrator rights on the target host.            |
| `domain_joined`           | The target host is a domain member; v1 supports workgroup machines only.       |
| `unreachable`             | WinRM cannot be contacted (TCP, TLS, auth, timeout, context cancellation).     |
| `machine_mismatch`        | Live `MachineGuid` differs from the state ID; the underlying machine changed.  |
| `concurrent_modification` | An external rename happened between the pre-flight read and `Rename-Computer`. |
| `unknown`                 | Catch-all for unmapped PowerShell or WinRM failures.                           |

## Permissions

- Local Administrator on the target Windows host.
- WinRM enabled and reachable (HTTP/5985 or HTTPS/5986).
- Read access to `HKLM:\\SYSTEM\\CurrentControlSet\\Control\\ComputerName\\*`
  and `HKLM:\\SOFTWARE\\Microsoft\\Cryptography`.

## Import

A `windows_hostname` resource is imported using the target machine's
`MachineGuid` (NOT the hostname — the hostname is not a stable identifier):

```shell
terraform import windows_hostname.this 11111111-2222-3333-4444-555555555555
```

Retrieve the `MachineGuid` on the target host with:

```powershell
(Get-ItemProperty -Path 'HKLM:\\SOFTWARE\\Microsoft\\Cryptography' -Name MachineGuid).MachineGuid
```
